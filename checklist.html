DUAL SIMULATOR

One file, two simulators

Each auto-detect item has both an X-Plane dataref and MSFS SimVar. The app uses the right one based on which sim is connected. Checklist authors fill in both — users don't have to think about it.

{
  "label": "Master Battery",
  "expected": "ON",
  "auto_detect": {
    "condition": "> 0",
    "xplane_dataref": "sim/cockpit/electrical/battery_on",
    "msfs_simvar": "ELECTRICAL MASTER BATTERY"
  }
}

When connected to X-Plane, it reads sim/cockpit/electrical/battery_on. When connected to MSFS, it reads ELECTRICAL MASTER BATTERY. Same condition (> 0) for both.

CREATE

Make a checklist in 5 minutes

01

Open the editor

Go to the visual editor. No install needed — it runs in your browser.

02

Add phases and items

Create phases (Pre-Flight, Engine Start, Taxi...). Add items with label, expected value, and how-to instructions. Drag to reorder.

03

Add auto-detect (optional)

For each item, add the X-Plane dataref and/or MSFS SimVar that corresponds to this action. Set the condition (e.g. > 0 for ON, == 4 for BOTH fuel selector).

04

Export JSON

Click "Export JSON". You get a clean file ready to use.

05

Share it

Import into the desktop app, or submit a Pull Request to cockpitflow-data so everyone can use it.
